部署步骤（以阿里云/腾讯云轻量应用服务器为例）                                                                
  1. 买服务器，选配置

  教育场景 GPU 不是必须的——你调的是各厂的 API，自己服务器只做转发和编排。建议：

  ┌──────┬──────────────────┐
  │ 配置 │       推荐       │
  ├──────┼──────────────────┤
  │ CPU  │ 2 核             │
  ├──────┼──────────────────┤
  │ 内存 │ 2-4 GB           │
  ├──────┼──────────────────┤
  │ 系统 │ Ubuntu 22.04     │
  ├──────┼──────────────────┤
  │ 带宽 │ 3-5 Mbps（够用） │
  └──────┴──────────────────┘

  国内云现在轻量应用服务器一年几十块到一百多。

  2. 把后端代码搬上去

  # 在服务器上
  git clone <你的仓库>
  cd render
  pip install -r requirements.txt

  3. 配环境变量

  export QWEN_KEYS="sk-xxx,sk-yyy"
  export GLM_KEYS="..."
  export DOUBAO_KEYS="..."
  export DEEPSEEK_KEYS="..."
  export API_TOKEN="ai_boss_2026"
  export PORT=8000

  4. 用 systemd 保活

  # /etc/systemd/system/ai-solver.service
  [Unit]
  Description=AI Solver Backend
  After=network.target

  [Service]
  Type=simple
  User=ubuntu
  WorkingDirectory=/home/ubuntu/render
  EnvironmentFile=/home/ubuntu/render/.env
  ExecStart=/usr/bin/python3 main.py
  Restart=always

  [Install]
  WantedBy=multi-user.target

  sudo systemctl enable --now ai-solver

  5. 前端改一行 URL

  已配置为阿里云服务器地址：

  const API_BASE = 'http://120.27.144.30:8000';

  如果后续绑了域名 + 配了 SSL（用 certbot + nginx 反代），就改成 https://你的域名。

  ---
  迁移后的代码状态

  已完成迁移，前端代码简化为：
  - 默认地址：http://120.27.144.30:8000
  - 启动时仅做单次版本检查（移除 4 次 8s 重试唤醒）
  - /health 检查仅 3 次快速重试（原 30-45 次轮询）
  - /solve 请求 2 次重试兜底网络抖动

  ---
  架构不变：桌面客户端 + 云端 API，服务器地址支持环境变量/配置文件覆盖。
